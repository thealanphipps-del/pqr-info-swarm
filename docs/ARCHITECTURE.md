# PQR Architecture: The Fabric and the Swarm

## Drawing the Line
The PQR system is divided into two distinct domains:

### 1. The PQR Info Fabric (The Memory)
The Fabric is a passive, content-addressable storage layer. It is the **Source of Truth**.
- **Layers 0-2**: Absolute Truth, Genesis, and Fabric.
- **Responsibility**: Persistence, Forensic Integrity, and Content-Addressing.
- **Mechanism**: CockroachDB + Vault for encrypted metadata.

### 2. The Hyperdevelopment Swarm (The Logic)
The Swarm is an active, autonomous execution layer. It is the **Actuator**.
- **Layers 3-7**: Swarm, Cognition, Execution, Governance, and Sovereignty.
- **Responsibility**: Ticket Resolution, Code Mutation, and Self-Healing.
- **Mechanism**: LLM Agents (Gemini Pro / Local Gemma) + Metaprogramming Loops.

## Forensic Lineage
Every ticket in the Fabric must have a parent lineage. This ensures that every decision made by the Swarm can be audited back to the Genesis Root, preventing "Hallucination Divergence."
